# 第一部分：Time River 项目白皮书（完全完整版）

**项目名称**：Time River (时光流)  
**版本**：v2.1  
**日期**：2026 年 4 月  
**定位**：全球首个融合东西方命理、基于 Solana 与 Arweave 的 AI 驱动去中心化生命资产协议  


## 一、项目概述

Time River 是一个将东西方命理智慧与 Web3 数据主权深度缝合的生命资产协议。它以 Solana 的高速低费与 Arweave 的永久存储为底层基础设施，允许用户将文字、影像、觉察记录等生命痕迹加密铸造为一条不可篡改的"时间之河"。平台内置的双命理 AI 引擎（八字十神 + 占星流年相位）可对用户过往记忆进行跨时空复盘分析，并支持将凝结了成长顿悟的"经验气泡"一键 Mint 为可交易的 NFT 或链上声誉凭证。

针对"不知道记什么"的普遍痛点，Time River 内置 **AI 记录灵感引擎**，根据用户命盘、当日运势、历史记录风格，每日推送个性化记录提示与引导模板，让每个人都能轻松发现平凡日子里的闪光点。

**核心理念**："记录即修行，觉察即改命"——让每一次记录都沉淀为可携带、可验证、可持续生息的链上身份资产。

**用户一句话感知**：Time River 是一个"永远不删日记的保险箱 + 懂命理又会翻旧账的贴心军师 + 能把人生阅历当资产卖的小金库 + 手把手教你记录生活的灵感向导"。


## 二、市场背景与定位

### 2.1 市场痛点

在 Web3 时代，数据不仅是信息，更是可量化的资产。然而人类最宝贵的资产——生命经验与自我觉察——往往被碎片化地遗忘在中心化平台的服务器中，既无法确权，也难以永久留存，更无法转化为实际价值。与此同时，全球玄学与身心灵市场正经历高速增长：据 Cognitive Market Research 数据，2025 年全球玄学市场营收高达 3920 亿美元，身心灵疗愈类 App 市场预计从 2024 年的 21.6 亿美元增长至 2033 年的 73.1 亿美元，年复合增长率约 14-15%。

**隐藏痛点：记录的门槛**  
大量用户放弃记录的原因是"我的生活太平凡，没什么值得写的"。这源于对"记录"的误解——并非只有惊天动地的大事才值得铭记，细微的情绪波动、一闪而过的念头、重复日常中的微小变化，都是构成个人生命图谱的重要像素。Time River 需要解决这个认知门槛，通过 AI 主动引导，将"记录"从负担变为乐趣。

### 2.2 竞品格局与差异化

当前市场存在三类相关竞品，但均未完全覆盖 Time River 的复合价值区间，更无一款产品系统性地解决"记录引导"问题。

**第一类：Web3 情感/记忆记录平台**  
- **SoulChain**：去中心化情感日记平台，用户可匿名记录悔恨、梦想、人生经验，形成不可篡改的人类情感档案。技术栈覆盖 Solana、BSC、Polygon、Base 等多链。缺陷：无命理元素，无资产化功能，无记录引导机制。  
- **DreamForge**：记录梦境并通过 AI 分析情绪/主题，将 SHA-256 哈希时间戳上链 Solana。缺陷：仅做哈希存证，功能单薄，无日常记录引导。  
- **Moodiary**：AI 分析日记情绪并推荐歌单，将日记 Mint 为 NFT。缺陷：侧重娱乐化，无存证链，无命理层，引导方式单一。

**第二类：Web3 时间/资产化平台**  
- **PIEVERSE**：定位"时间经济"平台，将时间转化为可交易资产。核心机制包括智能 AI 日历与 TimeFi 模块（时间挑战、时间出价等）。已获 700 万美元融资（Animoca Brands 和 UOB Ventures 领投）。缺陷：聚焦"日程时间"而非"生命经验"，更偏向任务激励型，无记录引导。  
- **Gradient**：链上 AI 记忆层，将人的著作、录音、思想在链上永久保存并转化为数字化身。缺陷：偏"数字永生"宏大叙事，技术门槛高，普通用户难以日常使用。

**第三类：AI 命理/玄学 Web3 平台**  
- **SUPERFORTUNE ($GUA)**：AI 驱动的 Web3 玄学平台，融合八字、易经、风水。由 Manta Network 孵化，提供每日运势、择吉决策、个性化答疑。用户超 5.4 万，日均活跃用户 1.5 万，累计交易超 110 万笔。缺陷：专注命理预测，无"经验记录/存储"功能，无资产化与记录引导。  
- **人生 K 线**：将八字命理与股市 K 线结合，AI 生成可视化"人生 K 线图"。基于 BSC 链，被央视网、光明日报等主流媒体报道。缺陷：轻量化工具，无永久存储和资产化，无记录功能。

**Time River 的差异化优势**：同时融合 **东西方双命理引擎 + Arweave 永久加密存储 + Solana 链上资产化（SBT/cNFT）+ RAG 跨时空 AI 复盘 + 个性化记录引导系统** 五个维度，构成当前市场的唯一解。

### 3.1 交互设计：生命长卷（Life Scroll）

- **记忆气泡**：记录以悬浮气泡形式呈现，支持文字、加密图片预览、视频缩略图及文件图标。点击时触发涟漪特效并完成解密。

**天时脉搏仪表盘（Destiny Pulse Widget）**  
- 常驻侧边的磨砂玻璃浮窗。  
- 实时显示：  
  - 八字流日十神（如：食神日、正官日、偏财日等）  
  - 星座实时相位（如：水逆、木星顺行、火星刑克土星、金星进入新宫位等）  
  - AI 每日温情微语（一句基于双命理的简短提示）

**记录引导区域（Inspiration Zone）**  
- 位于时间流顶部或空白处的醒目位置。  
- 展示"今日记录灵感"卡片：包含一条主要提示问题、2-3 个可选模板标签。  
- 灵感气泡：悬浮按钮，点击后随机显示一条记录提示，可刷新或直接使用。  
- "不知记什么？"按钮：点击打开 AI 对话引导流程。

### 3.2 功能模块一：AI 记录灵感引擎

这是解决"不知道记什么"痛点的关键设计。系统通过多维数据分析，为每位用户生成个性化的每日记录提示。

#### 3.2.1 灵感来源维度

| 维度 | 说明 | 示例 |
|------|------|------|
| **命理天时** | 根据当日流日十神、星座相位，提示适合记录的方向 | "今日食神透干，适合记录任何让你感到愉悦或满足的小事。" |
| **历史记录语义** | 分析用户过往记录的主题、情绪、高频词，推送相关延伸话题 | "你上周提到了'焦虑'，今天可以观察一下：焦虑感在什么时候最轻？" |
| **记录频率与时段** | 识别用户活跃时段与空窗期，适时推送轻量提示 | "你已经 3 天没记录了，今天只要写一句话描述此刻的心情就好。" |
| **用户命格特质** | 根据八字日主与五行喜忌，推送符合其性格的记录风格 | "身为乙木之人，你擅长感知细微情绪，试试记录今天一个让你心软的瞬间。" |
| **社会天象/节气** | 结合二十四节气、新月满月、水逆等集体事件，推送普适性话题 | "今日春分，昼夜平分，记录一件你想在新季节开始的事。" |

#### 3.2.2 记录引导形式

- **每日一问**：开屏或时间流顶部显示一句开放式问题，点击即创建新记录并预填问题。如："今天哪个瞬间让你觉得'活着真好'？"  
- **模板卡片**：针对不同场景提供结构化模板，用户点击后进入记录界面，按模板填写。
  - **感恩三事**：列出今天三件值得感谢的小事。
  - **情绪气象站**：用天气形容此刻心情，并简述原因。
  - **高光/至暗时刻**：记录一天中最强烈的一个正面/负面体验。
  - **对话未来**：给一年后的自己写一段话。
  - **命理觉察**：基于当日十神/相位的引导句式。如："今天正官日，我是否在某个场合承担了更多责任？"
- **灵感气泡**：在时间流空白处悬浮一个半透明气泡，点击后随机显示一条记录提示，可刷新或直接使用该提示创建记录。
- **AI 对话式引导**：用户点击"不知道记什么"按钮，AI 以对话形式通过 2-3 个问题帮用户挖掘素材，最终自动生成一段草稿供用户编辑确认。例如：
  - AI："今天有遇到什么人吗？"
  - 用户："见了老朋友。"
  - AI："和老朋友相处时，你的情绪和平时独处有什么不同？"
  - 用户回答后，AI 总结并生成一段记录草稿，用户可修改后保存。

### 3.3 功能模块二：多模态隐私记录与永存

- **前端加密**：数据在离开用户设备前，通过 Solana 钱包私钥派生的密钥进行 AES-256-GCM 加密。加密过程在浏览器端完成，私钥永不离开用户设备。  
- **双链存证**：加密文件通过 Irys（原 Bundlr）上传至 Arweave 获得 CID；记录的哈希值（SHA-256）与 CID 通过 Anchor 程序存入 Solana 链上账户，实现秒级存证与不可篡改。每条记录 Gas 成本极低（约 0.000005 SOL）。  
- **隐私分享升级**（远期）：集成 Lit Protocol 或 Arcium（基于 MPC+ZKP 的隐私计算网络），实现条件解密代理重加密。例如经验 NFT 售出后，买家自动获得解密权，卖家无需暴露密钥。同时支持"加密状态下的 AI 分析"，AI 输出洞察而不暴露原始内容。

### 3.4 功能模块三：东西方全知 AI 导师

- **八字引擎（Java）**：基于开源库 6tail/lunar 实现精密计算，涵盖四柱、十神、大运、流年、五行强弱分析。  
- **占星引擎**：基于瑞士星历（swisseph-wasm）实现行星位置、宫位、相位计算。Swisseph 基于 NASA JPL DE431 星历数据，覆盖公元前 13201 年至公元 17191 年。  
- **AI RAG 对话**：用户提问时，系统通过 pgvector 向量检索历史记忆（语义相似度搜索，可限定时间范围）；同时计算今日流日十神与流年影响，以及当前过运行星与本命盘的相位；将以上 Context 注入 Claude 3.5 Sonnet，生成兼具个人历史复盘与命理天时建议的个性化回复。  
- **远期个性化**：用户可对 AI 回复点击"准/不准"，数据积累后训练个人微型 LoRA（约 10MB），实现 AI 越用越懂用户的命理响应模式。

### 3.5 功能模块四：命书测算与动态修正

Time River 提供专业的八字命理排盘与多维度解读服务，并支持用户与 AI 持续对话修正，最终生成可存证、可交易的完整命理报告。

#### 3.5.1 基本信息采集与排盘
- **输入表单**：姓名（可选）、性别、出生日期（公历/农历切换）、出生时间（精确到分钟）、出生地点（省/市/区三级联动，用于真太阳时校准）。  
- **排盘计算**：后端基于用户出生信息计算：
  - 四柱八字（年柱、月柱、日柱、时柱）
  - 十神分布（比肩、劫财、食神、伤官、正财、偏财、正官、七杀、正印、偏印）
  - 五行旺衰与喜用神
  - 大运起运时间与每十年大运干支
  - 当前流年、流月、流日、流时运势

#### 3.5.2 AI 深度推演与命书生成
- **命书总纲**：AI 根据排盘结果生成开篇总论，概括命主性情、格局层次与人生大趋势。例如："此命生于寒冬，木火相生，性情如春，然内有戊土重重，乃为沉稳之主。"  
- **分类解读**：系统自动生成以下维度的详细解析：
  - **性格命格**：日主特性、五行气质、优缺点。
  - **事业功名**：适合行业、职场表现、晋升机遇与挑战。
  - **婚姻姻缘**：配偶特征、感情走势、相处建议。
  - **财富运势**：正偏财强弱、理财建议、投资时机。
  - **子女缘分**：子女数量倾向、亲子关系、教育建议。
  - **健康寿元**：体质偏性、易患疾病、养生重点。
  - **六亲关系**：与父母、兄弟姐妹、朋友的关系特征。
- **铁板神数/经典引用**：每段解读附有命理经典口诀或神数条文作为依据，增加专业感与可信度。  
- **准/不准反馈**：每段解析下方提供"准"或"不准"的反馈按钮，数据用于训练个人命理模型。  
- **对话修正**：用户可就任何一条解析与 AI 导师展开多轮对话，例如："我为什么总是情绪压抑？有没有具体的化解方法？"AI 结合八字原理与用户历史记录提供更个性化建议。

#### 3.5.3 报告生成与资产化
- **报告聚合**：用户满意后，可将经过修正的完整命书（含总纲+所有分类解读）一键生成为精美长图（Canvas 渲染）或 PDF 文档。  
- **链上存证**：报告生成的同时，系统自动将报告内容的哈希值与 Arweave CID 存入 Solana 链上，生成"命书存证记录"，不可篡改。  
- **命书 NFT**：用户可选择将完整命书铸造为专属 cNFT（Compressed NFT，基于 Metaplex Bubblegum 标准），作为个人数字身份藏品。命书 NFT 可设隐私选项（仅持有者可见全文，或公开摘要），未来可在二级市场交易（含版税）。

### 3.6 功能模块五：经验资产化系统

- **生命勋章（SBT，Soulbound Token）**：  
  - **触发条件**：首次记录、连续记录 7 天、连续记录 100 天、首次出售 NFT、获得一定数量的 AI 准确反馈、完成命书修正等。  
  - **用途**：勋章不可转让，构成用户在 Web3 世界的链上信用画像。远期将符合 W3C 可验证凭证（Verifiable Credential）标准，用户可选择将"连续记录天数""总记录条数"等匿名指标开放给其他 DApp（如借贷协议作为信用补充、DAO 治理作为权重依据）。  
- **经验 NFT（cNFT，可交易）**：  
  - **铸造玩法**：用户在"长卷"中选择一条或多条记录，点击"铸造成经验"，设置标题、简介、售价（可选），系统调用 Metaplex Bubblegum 创建压缩 NFT，元数据 URI 指向 Arweave 上加密内容的索引文件。单次铸造费用极低（约 0.0005 SOL）。  
  - **购买玩法**：买家在 Marketplace 购买 NFT 后，获得解密权限（远期通过 Lit Protocol 自动实现，当前 MVP 可手动发送解密密钥），可查看核心内容。每次转售，原作者钱包自动收到版税（建议比例 5-10%），无需人工干预。  
  - **用户收益**：直接出售经验获得 SOL/USDC；持续获得二级市场版税；早期铸造者可获得 $RIVER 代币激励。

### 3.7 功能模块六：周期能量报告

- **一键生成**：日/周/月/年报，自动聚合周期内的记录数据与命理参数。  
- **可视化指标**：  
  - 五行气场起伏曲线（基于每日五行能量变化绘制）  
  - 星象共鸣深度变化图（显示重要行运相位的影响强度）

## 四、技术架构

| 层级 | 技术选型 | 职责说明 |
|------|---------|----------|
| **区块链执行层** | Solana (Anchor Framework / Rust) | 身份认证、SBT 合约、cNFT 合约（Metaplex Bubblegum）、存证哈希记录 |
| **永久存储层** | Arweave (via Irys) | 加密后多媒体文件的永久存储与检索 |
| **前端** | Next.js 14 + Framer Motion + Solana Wallet Adapter | 水平时间流交互、视差动画、钱包连接、前端加密、命理信息表单、记录引导界面 |
| **后端服务层** | Spring Boot 3.2 | 八字排盘计算、命书生成、灵感提示生成、业务逻辑编排 |
| **数据库与向量层** | Supabase (PostgreSQL + pgvector) | 用户档案存储、记录元数据存储、向量化历史记忆检索、命书反馈数据、记录风格标签 |
| **AI 模型层** | Claude 3.5 Sonnet + Embedding API (text-embedding-3-small) | 跨时空深度复盘生成、记录向量化嵌入、命理报告内容生成与对话修正、记录灵感与引导对话生成 |
| **占星计算层** | swisseph-wasm / OpAstro | 行星位置、宫位、相位计算（可前端或后端执行） |

### 4.1 数据库设计要点

- **profiles 表**：存储钱包地址、八字排盘 JSON、星盘数据 JSON、AI 交互偏好、命书反馈历史、**记录风格标签（日主、五行喜忌、活跃时段偏好）**。  
- **records 表**：存储 arweave_cid、solana_tx_hash、文件类型、加密后的 AI 视觉描述、向量索引（1536 维）、**灵感来源 ID（若由提示生成）**。  
- **marketplace 表**：存储已 Mint 为 NFT 的经验资产元数据及版税逻辑。  
- **destiny_reports 表**：存储用户命书报告版本、各维度解读内容、反馈记录、报告存证 CID 与交易哈希。  
- **inspiration_prompts 表**：预置提示模板库，含维度标签（命理/历史/节气等）、日主适配权重、触发条件。

### 4.2 去中心化与不可篡改要求（硬性约束）

Time River 必须在架构层面严格遵循 Web3 核心原则：

1. 用户数据存储于 Arweave，支付一次永久存储，无中心化服务器可删除或修改。  
2. 每条记录的哈希与 CID 通过 Anchor 程序存入 Solana 主网，存证交易一旦确认即不可逆。  
3. 用户钱包私钥自托管，加密密钥由私钥派生，平台在任何环节都接触不到明文数据或私钥。  
4. NFT 与 SBT 资产直接铸造至用户钱包，元数据指向 Arweave CID，资产所有权完全归用户。  
5. 用户可无需平台许可，直接从 Arweave 提取加密文件并使用自有密钥解密，实现"完全退出权"。  
6. 远期 AI 推理需向可验证计算方向演进（集成 TEE 或去中心化推理网络），消除中心化 AI 信任假设。

## 项目定义
Time River 是全球首个融合东西方命理（八字+占星）、基于 Solana 与 Arweave 的 AI 驱动去中心化生命资产协议。

- 区块链执行层：Solana (Anchor/Rust) - 身份认证、SBT 合约、cNFT (Metaplex Bubblegum) 合约、存证哈希记录

  - 星座实时相位 (如：水逆、木星顺行、火土刑克、金星进入新宫位)

  - 位于时间流顶部或空白处的醒目位置，采用半透明毛玻璃卡片设计。

  - 「不知记什么？」按钮：点击打开 AI 对话引导流程，以聊天界面呈现。
- 命书测算界面：独立页面，包含以下区域：
  - 基本信息表单：姓名(可选)、性别、出生日期(公历/农历切换)、出生时间(时/分)、出生地点(省/市/区三级联动，用于真太阳时校准)。
  - 排盘结果展示：四柱八字、十神、五行旺衰、大运起运时间、当前流年流月。
  - AI 深度推演区域：命书总纲 + 七维度分类标签页 + 铁板神数引用 + 准/不准反馈按钮。
  - 对话修正栏：用户可就任一解析条目提问，AI 结合命理与历史记录给予细化回答，支持多轮对话。
  - 报告生成按钮：汇总所有内容生成长图/PDF，触发链上存证与 NFT 铸造。
- 周期能量报告：一键生成日/周/月/年报，包含五行气场曲线、星象共鸣深度、AI 关键词总结，通过 Canvas 渲染生成长图或直接铸造成报告 NFT。

## 详细功能模块与玩法说明

### 1. AI 记录灵感引擎（核心模块）

#### 1.1 灵感生成维度
- **命理天时**：根据用户当日流日十神、星座过运相位，提示适合记录的方向。
- **历史记录语义**：通过向量检索分析用户过往记录的主题、情绪倾向、高频词，推送延伸话题。
- **记录频率与时段**：识别用户活跃时段与空窗期，适时推送轻量提示。
- **用户命格特质**：根据用户八字日主与五行喜忌，匹配预设的记录风格提示。
- **社会天象/节气**：结合二十四节气、新月满月、行星逆行等集体事件，推送普适性话题。

#### 1.2 记录引导形式
- **每日一问**：开屏或时间流顶部显示一句开放式问题，点击即创建新记录并预填问题。
- **模板卡片**：感恩三事、情绪气象站、高光/至暗时刻、对话未来、命理觉察。
- **灵感气泡**：在时间流空白处悬浮一个半透明气泡，点击后随机显示一条记录提示，可刷新或直接使用。
- **AI 对话式引导**：用户点击"不知道记什么"按钮，AI 通过 2-3 个问题挖掘素材，最终自动生成一段草稿供用户编辑确认。

#### 1.3 因人而异的记录风格指南
系统在用户首次完成八字排盘后，生成一份《个人记录风格指南》，基于日主与五行配置提供记录建议。
日主-记录风格映射：甲木/乙木/丙火/丁火/戊土/己土/庚金/辛金/壬水/癸水（详见白皮书第 3.2.3 节）

#### 1.4 开发要点
- 灵感提示库需预先设计至少 50 条基础提示，按维度、日主适配、触发条件打标签存入 `inspiration_prompts` 表。
- 每日首次打开应用时，调用灵感推荐 API，综合用户日主、今日运势、历史记录语义向量、最近记录时间，返回 1 条主推荐 + 2-3 条备选。
- 用户使用某条灵感完成记录后，在 `records` 表中记录 `inspiration_id`，用于后续效果分析与推荐优化。
- AI 对话引导需实现为有状态的多轮对话，前端显示类似聊天界面，最终输出草稿。

### 2. 多模态隐私记录与永存

- **前端加密**：AES-256-GCM，密钥通过 HKDF（基于 SHA-256）从钱包签名消息中派生，私钥永不离开用户设备。
- **上传重试**：Irys 上传失败时自动重试 3 次，仍失败则 fallback 至 IPFS (Pinata) 并记录备用 CID。
- **存证合约**：Anchor 程序定义 `RecordAccount` 结构，包含 `owner: Pubkey`、`content_hash: [u8; 32]`、`arweave_cid: String`、`timestamp: i64`。

### 3. 东西方双命理 AI 导师

- 八字引擎：使用 6tail/lunar-spring-boot-starter 集成，支持农历公历转换、八字四柱、十神、大运流年计算。
- 占星引擎：前端使用 swisseph-wasm 计算，减轻后端压力；或后端调用 OpAstro API。
- RAG 检索：使用 pgvector 的余弦相似度搜索，每次检索返回 Top 5 最相关记录。
- Context 构造模板：将历史记录摘要、今日运势、命盘信息按固定格式拼接后传入 Claude。

### 4. 命书测算与动态修正

玩法流程：
1. 用户填写生辰信息（含真太阳时校准）。
2. 后端八字引擎计算完整排盘，返回四柱、十神、五行喜忌、大运、流年等。
3. 前端调用 Claude API，生成命书总纲 + 七维度详细解析（含铁板神数引用）。
4. 每个解析段落下方提供"准"和"不准"按钮，记录反馈到 profiles 表。
5. 用户可就具体内容向 AI 提问，支持多轮对话修正。
6. 用户满意后，Canvas 渲染长图 → Irys 上传 → Anchor 程序存证 → 可选铸造命书 cNFT。

开发要点：
- 八字排盘需支持真太阳时校正，根据出生地点经纬度调整出生时间。
- 命书生成 Prompt 需结构化，建议使用 JSON 格式返回各维度内容，便于前端渲染。
- 命书长图需设计精美，包含 Time River 品牌元素与用户命盘信息。使用 html2canvas 或 Canvas 原生绘制。
- 铁板神数/口诀库需预先整理至少 50 条常用条文，按维度分类。
- 命书生成需处理长文本渲染性能，建议使用 Web Worker 进行 Canvas 渲染。

### 5. 经验资产化系统

- 生命勋章 SBT：Anchor SBT 合约，mint 后设置 `freeze_authority` 为 None 以禁止转移。
- 经验 NFT (cNFT)：使用 Metaplex Bubblegum SDK，创建 Merkle Tree 并铸造 cNFT。版税通过 Metaplex Token Metadata 程序的 `royalty_basis_points` 字段设置。

### 6. 命理社区与创作者经济

- 创作者认证：管理员审核后赋予 `creator` 角色。
- 付费专栏：内容存储于 Arweave，访问权限通过 Lit Protocol 控制，购买后解密。
- 1v1 解读：预约系统 + 加密聊天室，结算通过 Solana 程序托管支付，完成后释放款项。

## 去中心化与不可篡改要求（硬性约束）
1. 用户数据存储于 Arweave，支付一次永久存储，无中心化服务器可删除或修改。
2. 每条记录的哈希与 CID 通过 Anchor 程序存入 Solana 主网，存证交易一旦确认即不可逆。
3. 用户钱包私钥自托管，加密密钥由私钥派生，平台在任何环节都接触不到明文数据或私钥。
4. NFT 与 SBT 资产直接铸造至用户钱包，元数据指向 Arweave CID，资产所有权完全归用户。
5. 用户可无需平台许可，直接从 Arweave 提取加密文件并使用自有密钥解密，实现「完全退出权」。
6. 远期 AI 推理需向可验证计算方向演进，消除中心化 AI 信任假设。

## 开发约束与注意点
- 所有数据在离开前端前必须加密，私钥永不离开用户设备。
- Arweave 上传需实现重试机制，失败时 fallback 至 IPFS (通过 Pinata)。
- 命理相关内容必须附带免责声明：「基于传统文化算法的心理洞察，仅供娱乐参考」。
- Framer Motion 动效必须保证 60fps，气泡点击事件需处理异步解密延迟。
- 命书生成需处理长文本渲染性能，避免 Canvas 绘制卡顿。建议使用 Web Worker 进行 Canvas 渲染。
- 真太阳时校准需集成地理位置 API 或内置经纬度数据库。可使用免费 IP 定位或手动选择城市。
- 记录引导功能需注重轻量化，避免让用户感到被强制或骚扰，所有提示均可关闭或跳过。在设置中提供「关闭记录提示」选项。
